Многопоточный C++ TCP сервер с закрытием всех соединений и очищении памяти при разрыве соединения и других аварийных ситуациях.
Небольшое пояснение по логике работы ответов: если от 1 клиента передается сообщение с каким-то id, по которому он уже
авторизован, выдается соответсвующее сообщение, если id сообщения привязан к другой сессии, аналогично.
Поле client_id составляется из шифрования строки body.
Поля логин и пароль проверяются просто по длине строки.
